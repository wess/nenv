A virtual environment for Nelua · macOS via Homebrew

Per-project Nelua environments, one command away.

nenv builds an isolated .venv for the Nelua programming language right inside your project. Clone, compile, and symlink the toolchain, then source activate to put a project-local nelua on your PATH — and deactivate to step back out.

Get started View on GitHub →
terminal 
# spin up an isolated Nelua toolchain for this project
$ cd ~/projects/raycaster
$ nenv .venv
$ source .venv/bin/activate
[nelua : .venv] $ which nelua
→ ~/projects/raycaster/.venv/bin/nelua
[nelua : .venv] $ deactivate
$
Features

Familiar virtualenv ergonomics, made for Nelua.

A single, dependency-light shell script that keeps each project's compiler in its own sandbox.

📦

Isolated by project

Each nenv .venv lives inside your project directory, so one repo's Nelua never collides with another's.

⚙️

Builds Nelua for you

Clones nelua-lang and runs make, then symlinks the freshly built nelua into the venv's bin.

🔌

Activate & deactivate

Source the generated activate script to prepend the venv's bin to PATH; deactivate restores everything.

🏷️

Prompt awareness

Your shell prompt shows [nelua : .venv] while active, so you always know which environment you're in.

🍺

Homebrew install

Tap and install on macOS in two commands — no runtime dependencies beyond a shell, git, and make.

🪶

Pure shell, tiny

A handful of Bash scripts — no daemon, no config files, nothing to learn. It just does the one thing.

How it works

From empty directory to working compiler.

One nenv invocation does the whole setup; sourcing the activate script swaps your environment in and out.

Name the venv

Run nenv .venv in your project. nenv creates the venv directory and a bin for it.

Build Nelua

It clones nelua-lang, compiles it with make, and symlinks nelua into the venv.

Activate

source .venv/bin/activate prepends the venv bin to PATH and relabels your prompt.

Deactivate

Call deactivate to restore the original PATH and prompt — your shell is back to normal.

Get started

Install on macOS, then create your first venv.

Tap and install via Homebrew, then run nenv inside any project directory.

terminal 
# 1. install (macOS)
brew tap wess/package
brew install nenv

# 2. create an isolated Nelua environment in your project
cd ~/to/your/sweet/sweet/project/directory
nenv .venv

# 3. activate it
source .venv/bin/activate
# [nelua : .venv] $  ← your prompt now shows the venv

# ...build with the project-local nelua, then step out
deactivate
